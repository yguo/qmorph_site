## 一、产品与商业模式

### Q1: QMorph 和传统 IVR（按键式语音菜单）有什么本质区别？

传统 IVR 是树形菜单——"按 1 预约，按 2 查询"——用户必须适应机器的逻辑。QMorph 是自由对话式 AI，用户用自然语言说"我下周二下午想看牙"就能完成预约，体验接近真人前台。

核心差异：

| 维度 | 传统 IVR | QMorph |
|------|---------|--------|
| 交互方式 | 按键 / 固定关键词 | 自由自然语言对话 |
| 意图理解 | 预设 5-10 个分支 | LLM 理解 30+ 意图 + 动态上下文 |
| 完成能力 | 仅路由转接 | 端到端完成预约、查询、确认 |
| 多语言 | 预录每种语言 | 实时自动识别并切换 |
| 上线时间 | 2-4 周定制 | 5 分钟自助配置 |

### Q2: 目标客户为什么不直接雇一个前台？

可以雇，但成本差距巨大且有覆盖盲区：

- **成本**：美国全职前台年薪+福利约 $46,800。QMorph 月费 $99-$299，年费不到人工的 8%。
- **可用性**：真人只覆盖 8 小时工作时间，QMorph 24/7 全天候，包括午休、节假日、深夜急诊来电。
- **漏接损失**：牙科诊所 40% 来电发生在非接听时段。法律事务所错过一个来电平均损失 $2,000-$8,000 案值。

QMorph 不是完全取代人，而是让人专注高价值工作，AI 处理重复性接线任务。

### Q3: 如果 AI 回答错了怎么办？会不会带来法律风险？

这是我们设计中的第一优先级。防线机制：

1. **硬约束层**——AI 架构层面禁止特定行为：不能给医疗诊断、不能给法律建议、不能在未验证身份前透露患者信息。这些不是"告诉 AI 不要做"，而是代码层面的拦截。
2. **合规声明**——法律行业场景，每通电话开头强制播放"我不是律师"的免责声明，不可跳过。
3. **升级机制**——置信度低于阈值自动转人工，不猜测不硬答。
4. **HIPAA 合规**——医疗场景全链路加密、BAA 协议、审计日志，符合联邦法规。
5. **幻觉率目标 < 1%**——通过 Red Team 测试 + CI/CD 质量门持续保障。

## 二、电话接入与号码迁移

### Q4: SMB 客户需要换电话号码吗？他们的老客户会不会找不到他们？

**完全不需要换号。** 终端客户拨打的始终是 SMB 原有的那个电话号码，完全无感知。

实现方式有两种：

**方案 A：运营商侧呼叫转移（推荐给 POC 阶段）**

SMB 在自己的电话运营商设置 Conditional Call Forwarding，把来电转接到 QMorph 的 Twilio/Telnyx 号码。设置极其简单，在美国大多数运营商只需拨一个代码：

- `*61*[QMorph号码]#` → 无应答时转接（运营商默认 15-30 秒）
- `*67*[QMorph号码]#` → 忙线时转接
- `*62*[QMorph号码]#` → 不可达时转接
- `*72*[QMorph号码]#` → 无条件转接全部来电

**方案 B：号码移植（长期方案）**

将原号码通过 Porting 迁移到 Twilio/Telnyx 平台，由 QMorph 完全托管。周期约 1-4 周，适合正式付费客户。

### Q5: 能不能让电话先响真人，真人不接再转 AI？

**可以，这是 Twilio/Telnyx 的原生能力，QMorph 完全支持。**

技术实现方式是 `<Dial>` TwiML 的 `timeout` 属性 + `action` 回调：

1. 来电进入 QMorph 平台后，先用 `<Dial timeout="N">` 拨打 SMB 的真人手机/座机
2. 真人在 N 秒内接了 → 双方正常通话，QMorph 不介入
3. 超时未接 → Twilio 回调 QMorph 的 action URL，返回 `DialCallStatus: no-answer`
4. QMorph 接管 → Voice Agent 接听来电

SMB 可以在 Dashboard 自由配置超时时间和路由策略：

| 时段 | 策略 |
|------|------|
| 上班时间 | 先响真人 20 秒，不接转 AI |
| 午休时间 | 先响真人 10 秒，不接转 AI |
| 下班 / 节假日 | 直接 AI 接听 |
| 日历显示全部排满 | 直接 AI 接听 |

这种"真人优先、AI 兜底"模式是 SMB 客户接受度最高的部署方式，也是 QMorph 的差异化卖点之一。

### Q6: 通话质量能保证吗？会不会有延迟或杂音？

QMorph 的通话走 PSTN → Telnyx/Twilio SIP Trunk → WebSocket 路径，通话质量与普通电话一致。

关于延迟（用户说完话到 AI 开始回复）：
- **目标 P50：~900ms，P95：<1500ms**
- 对比参考：真人前台平均思考+反应时间约 1-3 秒
- 实现手段：Deepgram Nova-3 (STT <300ms) + Claude 3.5 Sonnet (LLM <500ms) + Cartesia Sonic (TTS <200ms) 全链路流式处理

当网络质量差时，系统自动降级——降低音频码率、缩短回复长度，极端情况转为短信沟通。

## 三、技术架构与护城河

### Q7: 你们的核心技术壁垒在哪里？别人用同样的 API 能不能复制？

竞品确实可以用同样的 Deepgram + GPT-4 + ElevenLabs 组合搭一个 Demo。但 QMorph 的护城河在三个维度上层层递进：

**第一层：复杂场景下的超低延迟（最直接的用户感知壁垒）**

简单问答 800ms 很多人做得到。但在需要查日历、验保险、多轮确认的业务场景下仍维持 <800ms 感知延迟——这需要全链路系统级优化：

- **推测执行**——在 STT 还没结束时，基于中间转录结果预启动工具调用（类似 CPU 分支预测）
- **语义缓存**——对高频问答（占通话量 40-60%）直接命中缓存，跳过 LLM，延迟从 ~900ms 降到 <200ms
- **自适应管道调度**——根据实时网络质量和用户说话节奏动态调整全链路参数

每降低 100ms 都需要大量工程经验积累，这不是换一个更快的模型能解决的。

**第二层：行业知识图谱 + 硬约束引擎（准确率壁垒）**

- 垂直行业的 Slot-Filling 专用小模型（2B 参数），在信息提取准确率上超过通用 GPT-4o，延迟低 5 倍
- 架构级硬约束——不依赖 LLM "记住"规则，而是代码层面阻止违规行为
- 动态 Prompt 压缩——30 轮对话压缩到 200 token，零信息损失

**第三层：评估驱动的自动改进飞轮（数据壁垒）**

- 每通真实电话自动标注结果（成功/失败/原因），回流到模型优化
- A/B Testing 框架直接用真实通话对比 Prompt 变体的业务指标
- 失败模式自动挖掘，生成知识库补丁

竞品可以抄架构、用同样的模型，但无法复制通过百万通真实通话积累的行业数据和改进策略。

### Q8: 为什么选 Telnyx 而不是 Twilio？Twilio 不是更成熟吗？

两者都可以支持，QMorph 的架构是电话层可插拔的。但默认推荐 Telnyx，原因：

| 维度 | Telnyx | Twilio |
|------|--------|--------|
| 每分钟通话成本 | $0.002 | $0.0085 |
| WebSocket 媒体流延迟 | 更低（自建网络骨干） | 稍高 |
| 价格优势 | 便宜 **76%** | 行业标杆但贵 |
| STIR/SHAKEN 防骚扰标记 | 原生支持 | 支持 |
| 生态集成 | 相对小众 | 最丰富 |

在 POC 阶段或客户已有 Twilio 账户的情况下，也完全支持 Twilio。

### Q9: 如何保证 AI 不会"学坏"或被恶意利用？

QMorph 的安全防线包括四层：

1. **Prompt Injection 防御**——Red Team 测试覆盖注入攻击、角色扮演攻击、PHI 窃取、医疗建议绕过四大类向量，全部纳入 CI/CD 质量门。
2. **骚扰电话处理**——三级递进检测机制，严重辱骂自动结束通话并记录号码，24 小时内 3 次触发进入临时黑名单。
3. **对话边界硬约束**——AI 永远不会给出医疗诊断、法律建议、药物剂量。这些是架构层面的禁令，不是 prompt 层面的"请不要"。
4. **持续监控**——Langfuse 监控所有对话，异常模式自动告警，人工审核闭环。

## 四、部署与运营

### Q10: SMB 客户上线需要多久？需要技术人员吗？

**目标：5 分钟自助上线，零技术门槛。**

完整流程：

| 阶段 | 耗时 | 操作 |
|------|------|------|
| 注册 + 业务信息 | 3 分钟 | 通过多轮对话告诉 QMorph 业务类型、服务项目、营业时间 |
| Agent 自动生成 | 30 秒 | 系统自动构建 Voice Agent（Prompt + 音色 + 业务逻辑） |
| 测试拨打验证 | 1 分钟 | 点击"Test Call"试听效果 |
| 电话转接配置 | 1 分钟 | 按引导在手机上拨转接代码 |
| **上线** | — | 来电自动由 AI 接听 |

后续进阶配置（日历同步、CRM 集成、知识库完善）可在上线后逐步完成，不阻塞首次上线。

### Q11: 支持哪些语言？西班牙语市场怎么做？

目前支持 **英语、中文、西班牙语** 三种语言，通话中实时自动检测并切换。

西班牙语是 QMorph 的重要差异化方向——北美有 4,400 万西班牙语母语用户，但所有竞品的西班牙语支持都是"翻译级"而非"原生级"。QMorph 的目标是提供具备拉丁裔文化语境适配的原生体验，而不仅仅是翻译。

语言切换规则：
- 来电首句自动检测语言（置信度 > 80% 才切换）
- 用户随时可以说 "English please" / "Español" / "请说中文" 手动切换
- 切换后全程保持该语言
- 所有对话转录统一存储为英文（便于 SMB 查阅报告）

### Q12: 系统宕机了怎么办？来电会丢失吗？

不会。多层容灾设计：

1. **电话层**——如果 QMorph 服务不可达，Twilio/Telnyx 自动执行 Fallback：回拨 SMB 原号码（相当于电话直接打给真人）
2. **LLM 层**——如果主 LLM（Claude）超时，自动降级到规则引擎快速响应
3. **TTS 层**——如果主 TTS（Cartesia）超时，切换到备用 TTS（Azure Neural）
4. **通话中断**——60 秒内同号码回拨自动恢复上下文，无需重新开始
5. **监控**——P95 端到端延迟 > 1500ms 自动触发 PagerDuty 告警

SLA 目标：99.95% 可用性（年度宕机 < 4.4 小时）。

## 五、竞争格局

### Q13: 和 Bland.ai、Air.ai 这些 AI 原生竞品比，QMorph 赢在哪里？

| 维度 | Bland.ai / Air.ai | QMorph |
|------|-------------------|--------|
| 定位 | 通用 AI 电话平台 | **行业垂直**（牙科/法律/家政深度适配） |
| 上手门槛 | 需要开发者写代码 | **5 分钟 No-Code 自助上线** |
| 合规 | 无 HIPAA 原生认证 | **HIPAA + BAA 原生支持** |
| 系统集成 | 通用 CRM | **行业核心系统**（Dentrix / Clio / ServiceTitan） |
| 定价 | 按分钟计费，难以预测 | **固定月费，预算可控** |

一句话：Bland.ai 是"开发者的电话 API"，QMorph 是"SMB 老板 5 分钟就能用的 AI 前台"。

### Q14: 和 Smith.ai、Ruby Receptionists 这些人工+AI 混合竞品比呢？

| 维度 | Smith.ai / Ruby | QMorph |
|------|----------------|--------|
| 月费 | $235-$1,095 | $99-$299 |
| 并发能力 | 受限于人工数量 | **无限并发** |
| 可用时间 | 工作时间（部分提供 24/7） | **24/7 全天候** |
| 扩展性 | 需要招聘更多人 | **即时扩展** |
| 一致性 | 取决于接线员个人水平 | **100% 一致** |

Smith.ai 的 G2 评分 4.7/5 说明"人工+AI"体验确实好。但价格是 QMorph 的 3-10 倍，且无法真正无限并发。QMorph 的目标是用纯 AI 达到 Smith.ai 的服务质量，同时保持 10 倍的价格优势。

## 六、投资与里程碑

### Q15: 第一个可演示的 POC 需要多久？

**目标：4-6 周。**

| 周 | 里程碑 |
|---|--------|
| W1-2 | 单行业（牙科）端到端 Voice Agent：来电 → 多轮对话 → 预约写入 Google Calendar → 短信确认 |
| W3 | PSTN 接入（Telnyx/Twilio）+ 呼叫转接 + 真人优先/AI 兜底路由 |
| W4 | 管理 Dashboard 基础版（通话记录、日历视图、基本配置） |
| W5-6 | 10 家 Beta 诊所真实测试 + 评估报告 |

团队配置：AI/ML Engineer 1 人 + Backend Engineer 1 人 + Voice/Telephony Engineer 1 人。

### Q16: 数据飞轮什么时候开始转？

从第一通真实电话开始，但要产生明显的复合效应需要规模：

| 阶段 | 通话量 | 能力解锁 |
|------|--------|---------|
| 0-1,000 通 | 发现主要失败模式，修复 Top 10 问题 |
| 1,000-10,000 通 | 语义缓存命中率 > 40%，意图识别准确率 > 93% |
| 10,000-100,000 通 | 行业 Slot-Filling 微调模型训练数据充足 |
| 100,000+ 通 | 自动改进飞轮成型——新问题自动发现→自动修复→自动验证 |

**关键点**：这是纯 AI 竞品都需要迈过的门槛。先跑起来的人，数据壁垒会越来越深。
