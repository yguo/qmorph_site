## 一、为什么是日本？

| 维度 | 数据 |
|------|------|
| SMB 数量 | 中小企业约 358 万家，占全部企业 99.7% |
| SaaS 市场规模 | 2026 年 ≈ $13.8B，预计 2035 年达 $38.1B（CAGR 13.5%） |
| 云 / SaaS 渗透率 | 仅 34%，远低于全球均值 93 款 SaaS（日本仅 35 款） |
| 政府补贴 | 「デジタル化・AI導入補助金」2026 年公募已启动，最高补贴 3/4 成本 |
| 牙科诊所 | ≈ 66,843 家，每家诊所日均电话量集中在预约与变更 |
| 竞品空白 | IVRy（按键导航 IVR）月活 5 万+，但尚未出现"端到端 AI Voice Agent" |

**核心判断**：日本 SMB 电话场景仍高度依赖人工接听，且政府补贴降低了客户采购门槛。先发者可在补贴窗口期建立行业口碑。

## 二、产品功能扩展与演进

### 2.1 日语语音链路适配

| 组件 | 当前（北美） | 日本方案 | 备注 |
|------|-------------|---------|------|
| STT | Deepgram (en-US) | AmiVoice API / Google Chirp 3 (ja-JP) | AmiVoice 为日本市占率 No.1，支持 8kHz 电话信道 |
| TTS | Cartesia Sonic | CoeFont / SakuraSpeech / Google Cloud TTS | CoeFont 有 10,000+ 日语声音；SakuraSpeech 主打汉字重音准确 |
| LLM | Claude 3.5 Sonnet | Claude Opus 4.x / GPT-5.x | Claude 日语 MMLU ≈ 96.9%（vs 英语基线），表现优秀 |
| 电话 | Twilio (US) | Twilio JP / NTT CPaaS / Telnyx | NTT CPaaS 支持 050 号码，本土合规性更强 |

### 2.2 敬语（Keigo）系统

日本商务场景必须使用正确的敬语层级，否则会被视为"失礼"。Voice Agent 必须内建敬语策略。

**三层敬语框架**

| 层级 | 日语名 | 用途 | 示例 |
|------|--------|------|------|
| 尊敬語 | Sonkeigo | 抬高对方行为——面向客户 | 「ご予約をお入れいたします」 |
| 謙譲語 | Kenjougo | 降低自身行为——Agent 自指 | 「確認させていただきます」 |
| 丁寧語 | Teineigo | 通用礼貌语——です/ます | 「お電話ありがとうございます」 |

**落地做法**：
1. System Prompt 中明确敬语层级规则："你是诊所的 AI 受付（受付 = 前台），对来电者一律使用 Sonkeigo＋Teineigo 组合。"
2. 敬语质检 Gate：每轮生成后经过正则 / 小模型检查，拦截不含 `ます/ございます/いたします` 等敬语句尾的回复。
3. 测试集中加入敬语红线场景：用"上から目線（居高临下）"对话测试 Agent 是否会被带偏。

### 2.3 日本特有功能需求

| 功能 | 说明 |
|------|------|
| 曜日指定 | 日本预约习惯按"星期"而非具体日期，需支持"来週の水曜日" 式解析 |
| 保険証確認 | 牙科/医疗场景需确认患者保险种类（社保/国保/自费） |
| 初診/再診区分 | 首诊与复诊流程不同，影响时间段分配 |
| 休診日自動案内 | 自动播报休息日与替代联系方式 |
| お盆/年末年始 | 日本特有长假期间的特殊话术与排班处理 |
| 診察券番号 | 患者凭"诊察券号码"进行身份确认，类似 Patient ID |

### 2.4 产品路线图

```
Phase 0 (Month 0-2)    日语 STT/TTS 集成 + 敬语 Prompt + 本地号码  
Phase 1 (Month 2-4)    牙科垂直场景：预约/取消/变更/初诊问诊  
Phase 2 (Month 4-6)    整骨院/美容サロン 扩展 + LINE 通知集成
```

### 3.2 云基础设施

| 选项 | 特点 | 推荐场景 |
|------|------|---------|
| AWS Tokyo (ap-northeast-1) | 最成熟、生态最完整 | 默认首选 |
| さくらクラウド (Sakura Cloud) | ISMAP 认证、政府采购友好 | 政府补贴项目 |
| GCP Tokyo | Chirp 3 + Vertex AI 原生集成 | 全 Google 技术栈 |
| Azure Japan East | Azure OpenAI Service 本地部署 | 使用 GPT 系列时 |

**关键原则**：语音数据必须存储在日本国内（APPI 要求），推荐 AWS Tokyo + S3 JP 或 Sakura Cloud。

### 3.3 合规要件（APPI）

| 要件 | 要求 | QMorph 对策 |
|------|------|------------|
| 数据本地化 | 语音/视频/短信数据须存于日本国内 | 部署 AWS Tokyo，录音存储 S3 JP |
| 跨境传输 | 需取得用户明示同意或传输至"适格国" | LLM 推理可在日本/美国，需披露 |
| 利用目的特定 | 个人信息收集前须明示用途 | 通话开头加入利用目的说明话术 |
| 安全管理措施 | 技术 + 组织 + 物理三重防护 | TLS 1.3 + 加密存储 + 访问日志 |
| 泄露通知 | 发生泄露须向 PPC + 本人通知 | 自动检测 + 72 小时通知流程 |
| 罚则 | 法人最高 ¥1 亿（≈ $70 万） | 定期内审 + 外审 |

## 四、种子用户招募：日本本土策略

### 4.1 目标行业优先级

| 优先级 | 行业 | 理由 |
|--------|------|------|
| P0 | 歯科（牙科） | 66,843 家，电话预约为主，IVRy 已验证市场教育 |
| P1 | 整骨院/接骨院 | 48,000+ 家，几乎全靠电话预约 |
| P1 | 美容サロン | 理发/美甲/美容院，预约变更频繁 |
| P2 | 法律事務所 | 高客单价但决策链长，适合中后期 |
| P2 | 不動産仲介 | 来电查询多，周末/晚间漏接严重 |

### 4.2 日本特有的 GTM 渠道

**渠道 1：歯科医師会（牙科医师协会）合作**

日本各都道府県都有歯科医師会（例：東京都歯科医師会、大阪府歯科医師会）。通过协会推荐可快速获得信任背书。

- 行动：联系协会事务局，申请在定期勉強会（学习会）上做 15 分钟 Demo
- 目标：每次勉強会获得 5-10 个深度意向

**渠道 2：レセプトソフト（诊疗报酬软件）合作伙伴**

日本牙科诊所普遍使用 レセプトソフト（如 Dentis、ORCA）进行医保报账。与这些厂商做 API 集成或联合推广可触达存量客户。

**渠道 3：IT導入補助金 登録ベンダー**

注册成为「IT導入支援事業者」，QMorph 被纳入补助金认定 IT 工具清单后，SMB 采购时可报销最高 75% 费用。

- 行动：向中小企业基盤整備機構（SMRJ）提交工具注册申请
- 时间窗口：2026 年度公募 3 月 30 日已开放

**渠道 4：Google マイビジネス（GMB）+ 口コミ**

日本用户极度依赖 Google 口碑评分。为试点客户提供"口コミ投稿テンプレート"（评价模板），通过 GMB 评分带来自然流量。

**渠道 5：LINE 公式アカウント 联动**

LINE 在日本月活 9,600 万（渗透率 > 80%），是"国民 App"。支持 LINE 消息通知（预约确认、提醒、变更）是刚需而非加分项。

### 4.3 种子用户招募 SOP（8 周）

| 周期 | 动作 | 产出 |
|------|------|------|
| Week 1-2 | 建立 200 家牙科诊所目标名单（Google Maps + 歯科医院ナビ） | 目标名单 CSV |
| Week 2-3 | 冷启动 50 通 DM（FAX / メール / LINE） | 15-20 家回复 |
| Week 3-4 | 提供无料体験（免费试用 14 天） | 8-10 家进入试点 |
| Week 4-5 | 上门拜訪（訪問）+ 现场 Demo | 深度关系建立 |
| Week 5-6 | 试点运行，收集通话录音 + NPS | 产品迭代依据 |
| Week 6-7 | 产出"导入事例"（成功案例）——含具体数字 | 可公开案例 2-3 篇 |
| Week 7-8 | 通过 Case Study 做二次传播（协会/口コミ） | 第二批 10 家意向 |

> **日本特色注意事项**：
> - **FAX 仍然有效**：日本中小诊所大量使用 FAX 接收营销材料，一页纸精美 FAX DM 的打开率远高于邮件
> - **上门 > 线上**：日本 B2B 销售极重视"面谈"，Video Call 成交率远低于面谈
> - **先生（Sensei）称呼**：对医生/律师必须使用"〇〇先生"而非"〇〇さん"

### 4.4 定价与补贴策略

| 套餐 | 月额 | 定位 |
|------|------|------|
| ライト（Light） | ¥9,800 (≈ $65) | 个人诊所 / 试用 |
| スタンダード（Standard） | ¥29,800 (≈ $200) | 中型诊所 / 整骨院 |
| プロ（Pro） | ¥59,800 (≈ $400) | 多拠点 / 法律事務所 |

- 首 14 天 **無料トライアル**（免费试用），降低决策门槛
- 引导客户通过 IT 導入補助金报销，实际到手价格可降至 1/4
- 提供年払い（年付）优惠 2 ヶ月分無料（免 2 个月）

### 6.4 法务与雇佣形态

| 选项 | 特点 | 推荐阶段 |
|------|------|---------|
| **業務委託（Business Contract）** | 无需设立法人，灵活 | 0-3 个月验证期 |
| **EOR（Employer of Record）** | 通过 Deel/Remote 等雇佣，合规且快速 | 3-6 个月 |
| **GK（合同会社）设立** | 注册资本 ¥1 起，设立费 ≈ ¥10 万 | 6 个月以上长期运营 |
| **KK（株式会社）设立** | 正式法人，客户信任度最高 | 年营收稳定后 |

**推荐路径**：先用業務委託快速起步 → 3 个月后用 EOR 雇佣 → 半年后设立 GK 法人。
